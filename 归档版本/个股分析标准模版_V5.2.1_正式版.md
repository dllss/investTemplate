## 第一章：数据核查规范 ⚠️ 【强制性前置】

> **铁律：未经核查的数据不得用于投资决策。数据错误，后续分析无论多精彩都是空中楼阁。**

### 1.1 数据核查原则

#### 核心原则
| 原则 | 说明 | 违规后果 |
|------|------|---------|
| **一手优先** | 优先使用公司财报原文，其次为交易所公告，最后才是金融终端 | 二手数据错误率>15% |
| **时点明确** | 每个数据必须标注时间点（股价日期、财报截止日期） | 过时数据导致估值误判 |
| **交叉验证** | 关键数据必须2个以上独立来源验证 | 单一来源错误无法发现 |
| **手工验算** | 衍生指标（股息率、PE、PB）必须手工计算，不直接引用软件 | 软件数据未更新/错误 |
| **误差容忍** | 允许±5%误差，超出必须重新核实 | 误差累积导致决策偏差 |

#### 数据源分级
| 级别 | 数据源 | 可信度 | 适用场景 |
|------|--------|--------|----------|
| **S级** | 公司年报/季报原文（PDF）、交易所公告 | 100% | 净资产、派息金额、股本 |
| **A级** | 官方投资者关系网站、港交所披露易、SEC EDGAR | 95% | 实时股价、重大事件 |
| **B级** | 付费金融终端（Bloomberg、Wind、FactSet） | 90% | 财务数据、历史数据 |
| **C级** | 免费财经网站（东方财富、雪球、Yahoo Finance） | 80% | 快速参考，需二次核实 |
| **D级** | 社交媒体、论坛、自媒体 | 60% | 仅作线索，必须核实 |

### 1.2 关键数据核查清单

#### 股价数据核查
| 检查项 | 标准 | 核查方法 | 常见陷阱 |
|--------|------|----------|----------|
| **最新价** | 当日收盘价 | 港交所/交易所官网 | 盘后交易价格混淆 |
| **货币单位** | 明确港元/人民币/美元 | 股票代码后缀(.HK/.SS/.US) | H股与A股价格混淆 |
| **复权处理** | 分红送股后需前复权 | 对比复权/不复权K线 | 股息率计算用错价格 |
| **成交量** | 日均成交额 | 近20日平均 | 流动性陷阱（小盘股） |

#### 财务数据核查
| 检查项 | 来源 | 验算公式 | 误差阈值 |
|--------|------|----------|----------|
| **年度派息** | 年报"拟派末期股息" | 每股派息×总股本=总派息额 | ±2% |
| **净利润** | 年报"归母净利润" | 扣非前后需区分 | ±3% |
| **净资产** | 年报"归母净资产" | 总资产-总负债-少数股东权益 | ±1% |
| **净现金** | 年报"现金及等价物"-"有息负债" | 需扣除受限资金 | ±5% |
| **总股本** | 年报"已发行股份数目" | 注意库存股 | ±0.1% |

#### 衍生指标手工验算
**必须手工计算，禁止直接引用软件数据：**

```python
# 股息率计算（以建行H为例，2026.02.24）
年度派息 = 0.40 元人民币/股  # 来源：2024年报
当前股价 = 6.62 港元         # 来源：2026.02.24收盘价
汇率     = 0.92 (人民币/港元) # 来源：当日中间价

股息率 = (年度派息 × 汇率) / 当前股价 × 100%
      = (0.40 × 0.92) / 6.62 × 100%
      = 0.368 / 6.62 × 100%
      = 5.56%

# 验证：若软件显示7.8%，则软件数据过时（基于2024年初股价4.5港元）
```

```python
# PB计算
每股净资产 = 9.25 元人民币    # 来源：2024Q3季报
当前股价   = 6.62 港元
汇率       = 0.92

PB = 当前股价 / (每股净资产 × 汇率)
   = 6.62 / (9.25 × 0.92)
   = 6.62 / 8.51
   = 0.78倍

# 注意：若用历史净资产计算，需标注"基于2024Q3数据"
```

```python
# 净现金/市值比（烟蒂股关键指标）
现金及等价物 = 12.6 亿港元  # 来源：2024半年报
有息负债     = 2.55 亿港元 # 来源：2024半年报，需区分长短债
净现金       = 10.05 亿港元
总股本       = 1.0 亿股
当前股价     = 5.51 港元
市值         = 5.51 亿港元

净现金/市值 = 10.05 / 5.51 = 1.82倍
# 结论：未达到2倍标准，需标注"接近烟蒂标准"
```

### 1.3 数据标注标准

#### 强制标注格式
所有分析中引用的数据必须按以下格式标注：

```
[指标名称]: [数值] ([来源], [日期], [验算状态])

示例：
股息率: 5.56% (基于2024年报派息0.40元及2026.02.24收盘价6.62港元, 已验算)
PB: 0.48倍 (基于2024Q3净资产9.25元及2026.02.24股价, 已验算)
净现金: 10.05亿港元 (来源：2024半年报, 已扣除有息负债2.55亿)
```

#### 数据时效性分级
| 时效标签 | 说明 | 使用限制 |
|----------|------|----------|
| **实时** | 当日收盘数据 | 可用于交易决策 |
| **近期** | 1周内数据 | 可用于趋势判断 |
| **滞后** | 1个月以上 | 需标注"可能存在偏差" |
| **过时** | 财报季前数据 | 禁止用于精确估值 |

#### 验算状态标签
| 状态 | 含义 | 可信度 |
|------|------|--------|
| **已验算** | 手工计算并交叉验证 | 高 |
| **待核实** | 引自单一来源，未手工验算 | 中 |
| **估算** | 基于历史数据推算 | 低 |
| **假设** | 情景分析用 | 仅作参考 |

### 1.4 流派分析前的数据核查流程

**所有标的进入四流派分析前，必须通过以下核查：**

```
开始分析标的：[股票名称] [代码]
    ↓
步骤1：获取基础数据
    - 最新股价（交易所官网，日期：____）
    - 最近完整财报（年报/半年报，日期：____）
    - 派息记录（最近年度，每股：____）
    ↓
步骤2：手工验算关键指标
    □ 股息率 = 派息/股价 = ____% (要求误差<±5%)
    □ PE = 股价/每股收益 = ____倍
    □ PB = 股价/每股净资产 = ____倍
    □ 净现金 = 现金-有息负债 = ____亿
    ↓
步骤3：交叉验证
    □ 对比Bloomberg/Wind数据（差异<±5%？）
    □ 对比同花顺/东方财富（差异<±5%？）
    □ 核对财报原文（关键数据是否一致？）
    ↓
步骤4：标注与存档
    - 数据时间戳：____
    - 验算人：____
    - 数据来源：____
    - 置信度：高/中/低
    ↓
通过核查 → 进入流派分类分析
未通过核查 → 退回补充数据，标记"数据不足"
```

### 1.5 常见数据陷阱与规避

#### 陷阱1：过时股息率
| 场景 | 错误 | 正确 |
|------|------|------|
| 建行H | "股息率7.8%"（基于2024年初股价4.5港元） | "股息率5.56%（截至2026.02.24，股价6.62港元）" |
| 影响 | 高估收益预期2.2% | 真实收益预期 |

#### 陷阱2：货币单位混淆
| 场景 | 错误 | 正确 |
|------|------|------|
| 蒙牛 | PE 12倍（未注明币种） | PE 12倍（港元计价，折合人民币需调整） |
| 中海物业 | 市值100亿（未注明） | 市值100亿港元 |

#### 陷阱3：净现金计算错误
| 场景 | 错误 | 正确 |
|------|------|------|
| 天津发展 | "净现金=市值2倍"（未扣负债） | "净现金=现金5亿-负债0亿=5亿，市值20亿，净现金/市值=0.25倍" |
| 影响 | 误判为烟蒂股 | 实际不符合烟蒂标准 |

#### 陷阱4：派息率误解
| 场景 | 错误 | 正确 |
|------|------|------|
| 中石化 | "派息率60%"（指分红比例） | "派息承诺60%，但需用净利润计算实际股息" |

#### 陷阱5：非经常性损益干扰
| 场景 | 错误 | 正确 |
|------|------|------|
| 长和 | "净利润-40%"（含一次性减值） | "经常性利润稳定，一次性减值104亿非现金" |

### 1.6 数据错误处理机制

#### 发现错误后的流程
```
发现数据错误
    ↓
立即标记："数据存疑，暂停决策"
    ↓
24小时内完成：
    1. 重新核实原始数据源
    2. 手工重新计算
    3. 更新所有相关分析
    4. 记录错误原因（过时/混淆/计算错误）
    ↓
修正后标注："[2026.02.24修正]原数据错误，已更新"
    ↓
评估影响：
    - 若影响流派归类（如金龟→银龟）→ 重新评估仓位
    - 若影响买卖点 → 调整交易计划
    - 若已执行交易 → 评估是否纠错
```

#### 责任追溯
| 错误类型 | 责任人 | 处理措施 |
|----------|--------|----------|
| 数据过时（>1个月） | 分析者 | 更新数据，降低该来源权重 |
| 计算错误 | 分析者 | 重新验算，标记"手工复核" |
| 来源混淆（A股/H股） | 分析者 | 建立检查清单，双重确认 |
| 财报误读 | 分析者 | 学习会计准则，咨询专业人士 |

## 第二章：流派识别与分流

### 2.1 初筛分流矩阵（基于验证后数据）

> **前提：以下所有标准数据必须经过第一章核查流程验证**

| 筛选维度 | 🛡️ 纯硬收息型 | 🔍 价值发现型 | 🔗 关联方资源型 | 🚬 烟蒂股 |
|---------|-------------|-------------|----------------|----------|
| **股息率门槛** | **>7%**（港股）<br>>5%（美股） | 3-6% | 3-5% | 0-3% |
| **PE区间** | <8倍 | 8-18倍 | 12-18倍 | 亏损或<5倍 |
| **PB区间** | <0.6倍 | 0.6-1.5倍 | 1.5-3倍 | **<0.3倍** |
| **净现金要求** | >市值30% | 有即可 | 轻资产为正 | **>市值×2倍** |
| **数据验证要求** | 派息需连续5年+<br>股价需近1周内 | FCF需TTM计算<br>净现金需扣除受限 | 母公司数据需核实<br>关联交易额需确认 | 现金需现金及等价物<br>非应收账款 |

**分流规则（数据验证后）**：
```
if 净现金>市值×2倍 and PB<0.3 and 有息负债接近零:
    优先归类 = 🚬 烟蒂股
    [需附：净现金计算表，现金真实性证明]
    
elif 股息率>7%(港股) and 连续5年分红 and 派息率30-70%:
    优先归类 = 🛡️ 纯收息
    [需附：派息历史表，派息率计算]
    
elif 轻资产 and 母公司为行业龙头 and PE<15 and 关联交易额<70%:
    优先归类 = 🔗 关联方资源型
    [需附：母公司土储/资源表，关联交易定价对比]
    
elif (EV/FCF<12倍) or (控股折价>30%) or (品牌PE<18且ROE>15%):
    优先归类 = 🔍 价值发现型
    [需附：FCF计算表或控股折价计算表]
    
else:
    进入观察区
    [需附：数据不足之处说明]
```
