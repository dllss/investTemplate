## 第一章：数据核查与AI预处理 ⚠️ 【强制性前置】

> **铁律**：未经核查的数据不得用于投资决策。AI计算结果必须经过人工验证。

### 1.1 AI辅助标签体系（新增）

引入 **[LLM-REQUIRED]** 和 **[COMPUTABLE]** 标签，明确人机分工：

| 标签 | 含义 | AI任务 | 人工任务 |
|------|------|--------|----------|
| **[COMPUTABLE]** | 可计算指标 | 抓取数据、计算比率、生成表格 | 验证数据源、检查异常值 |

### 3.2 现金流瀑布分析（新增，来自龟龟策略）

**[COMPUTABLE] + [LLM-REQUIRED]**

分析企业现金流的来源与去向：

```
经营现金流入
    ├── 核心业务收入 [COMPUTABLE]
    ├── 应收账款变化 [COMPUTABLE]
    └── 预收款项变化 [COMPUTABLE]
    ↓
经营现金流出
    ├── 采购支出 [COMPUTABLE]
    ├── 员工薪酬 [COMPUTABLE]
    └── 税费支出 [COMPUTABLE]
    ↓
经营现金流净额 [COMPUTABLE]
    ↓
投资现金流
    ├── 资本开支（维护性/扩张性）[LLM-REQUIRED区分]
    ├── 并购支出
    └── 理财/存款变化
    ↓
筹资现金流
    ├── 分红派息 [COMPUTABLE]
    ├── 债务偿还
    └── 股权融资
    ↓
可支配现金结余 [COMPUTABLE]
```

**关键指标**：
- **经营现金流/净利润** > 100%（说明利润有现金支撑）
- **维护性资本开支/经营现金流** < 50%（轻资产特征）
- **自由现金流（FCF）** = 经营现金流 - 维护性资本开支

### 4.2 SOTP分部估值（新增，来自龟龟策略）

**[COMPUTABLE] + [LLM-REQUIRED]**

适用于**控股平台型（D类）**和**多元化业务企业**。

#### 步骤1：业务分拆识别 [LLM-REQUIRED]

```
以华润医药为例：
├── 华润三九（上市，持股63%）
├── 华润双鹤（上市，持股60%）
├── 华润江中（上市，持股43%）
├── 东阿阿胶（上市，持股32%）
└── 分销业务（非上市）
```

#### 步骤2：各分部估值 [COMPUTABLE]

| 分部 | 估值方法 | 持股比例 | 归属市值 | 数据来源 |
|------|---------|---------|---------|---------|
| 华润三九 | 市值法 | 63% | 市值×63% | 实时股价 |
| 华润双鹤 | 市值法 | 60% | 市值×60% | 实时股价 |
| 华润江中 | 市值法 | 43% | 市值×43% | 实时股价 |
| 东阿阿胶 | 市值法 | 32% | 市值×32% | 实时股价 |
| 分销业务 | PE法（8-10倍） | 100% | 净利润×PE | 财报数据 |
| **合计** | **SOTP估值** | - | **分部市值加总** | - |

#### 步骤3：控股折价分析 [LLM-REQUIRED]

```python
# [COMPUTABLE]
控股折价率 = (SOTP估值 - 当前市值) / SOTP估值 × 100%

以华润医药为例（假设）：
SOTP估值 = 280亿港元
当前市值 = 170亿港元
控股折价率 = (280 - 170) / 280 = 39%

判断标准：
- 折价率 > 40%：深度折价，具备安全边际
- 折价率 20-40%：正常折价
- 折价率 < 20%：折价不足，等待回调
```

#### 步骤4：时间轴与催化剂 [LLM-REQUIRED]

| 时间 | 事件 | 影响 | 概率 |
|------|------|------|------|
| 2025H2 | 国企改革深化 | 可能提升派息率 | 中等 |
| 2026 | 子公司业务整合 | 提升协同效率 | 低 |
| 持续 | 回购注销 | 支撑股价 | 高 |

### 4.3 V3.5 FCEV估值法（保留+增强）

```python
# [COMPUTABLE]
EV = 市值 - NetCash（扣除有息负债后的净现金）
FCF = 经营现金流净额 - 维护性资本开支 [LLM-REQUIRED区分维护性/扩张性]
EV/FCF = EV / FCF

估值对照（保留V5.2.1标准）：
| EV/FCF | 评级 | 操作 |
|--------|------|------|
| <8倍 | 🐢🍊 金龟 | 重仓买入 |
| 8-10倍 | 🐢💎 银龟 | 分批建仓 |
| 10-12倍 | 🐢💰 铜龟 | 持有/观望 |
| >16倍 | 不符合 | 等待回调 |
```

## 第七章：穿透回报率验证（新增）

> **融合来源**：龟龟策略核心方法论
> **目的**：通过Top-Down和Bottom-Up双向验证，确保回报率计算准确

### 7.1 Top-Down回报率计算 [COMPUTABLE]

从**股东回报角度**计算：

```python
# [COMPUTABLE]
股东回报率 = (分红 + 回购 - 融资) / 期初市值 × 100%

或

股东回报率 = 净利润增长率 + 股息率 + 估值变化贡献
```

### 7.2 Bottom-Up回报率计算 [COMPUTABLE]

从**资产回报角度**计算：

```python
# [COMPUTABLE]
ROE = 净利润 / 净资产

可持续增长率 = ROE × (1 - 派息率)

预期回报率 = 可持续增长率 + 股息率
```

### 7.3 双向验证与差异分析 [LLM-REQUIRED]

| 情景 | Top-Down | Bottom-Up | 差异分析 | 结论 |
|------|---------|-----------|---------|------|
| **一致** | 12% | 11% | <2%差异 | ✅ 可信度较高 |
| **不一致** | 15% | 8% | >5%差异 | ⚠️ 需查明原因 |

**常见差异原因** [LLM-REQUIRED]：
- 一次性收益/损失（非经常性损益）
- 会计政策变化
- 融资活动影响
- 估值周期不同

| 🔗 关联方资源 | PE估值 | **时间轴分析** | PE<12倍 |
