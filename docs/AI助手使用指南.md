## 项目特点

### 为什么这个项目适合 AI 辅助？

这个项目是**知识管理型项目**，有明确的：

1. **标准化模板**（10章分析框架）
2. **严格的数据规范**（S/A/B/C/D 级数据源）
3. **清晰的文件结构**（07-分析输出、08-决策追踪）
4. **详细的 AI 指南**（AGENTS.md）

```
人类：提供投资需求和判断
    ↓
AI：按标准模板执行分析
    ↓
产出：标准化的投资分析报告
```

## AI 可以做什么

### 1. 生成投资分析报告 ⭐⭐⭐⭐⭐

**你说**：
```
帮我分析一下工商银行（01398.HK）
```

**AI 会**：
1. 读取 `个股分析标准模版.md` 获取分析框架
2. 查找工商银行的年报数据（S级数据源）
3. 按照10章框架逐一分析
4. 生成报告：`07-分析输出/工商银行_01398_投资分析报告.md`
5. 更新相关配置文件（侧边栏、首页列表）

**输出格式**：
```markdown
# 工商银行 (01398.HK) 投资分析报告

**一句话结论**：🐢🍊 极品金龟，PB 0.4倍，股息率8%，FCF倍数2.5倍，建议买入

## 第一章：数据核查
- 公司全称：中国工商银行股份有限公司
- 报表期：2023年年报
- 数据来源：2023年年报原文（S级数据源）
...
```

### 2. 更新模拟持仓数据

**你说**：
```
帮我记录买入 10000 股京投交通科技，价格 0.365
```

**AI 会**：
1. 更新 `simulation_trades.csv`（交易记录）
2. 更新 `simulation_state.json`（当前状态）
3. 更新 `dashboard_snapshot.json`（网页数据）×2 个位置
4. 验证数据一致性

### 3. 运行自动化脚本

**你说**：
```
帮我运行价格追踪脚本
```

**AI 会**：
```bash
cd 07-标的追踪
python3 auto_stock_tracker.py
```

### 4. 数据核查和验证

**你说**：
```
检查模拟持仓数据是否一致
```

**AI 会**：
```bash
python scripts/validate_simulation_data.py
```

### 5. 更新网站内容

**你说**：
```
把工商银行的分析报告添加到侧边栏
```

**AI 会**：
1. 编辑 `.vitepress/config.mjs`
2. 添加到 `sidebar` 配置
3. 更新 `07-分析输出/index.md`
4. 更新 `index.md` 首页列表

## 使用方式

### 方式 1：在 Cursor 中使用（你正在用）✅

**优势**：
- 已经在项目中，直接对话
- 可以直接编辑文件
- 可以运行终端命令

**使用方法**：

```
你：@AGENTS.md 帮我分析工商银行
    ↓
Cursor AI 读取 AGENTS.md 了解项目规范
    ↓
按照标准流程生成分析报告
```

### 方式 2：Claude Desktop（桌面应用）

**步骤 1：配置 MCP**

打开配置文件：
```bash
# macOS
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows  
%APPDATA%\Claude\claude_desktop_config.json
```

**步骤 2：添加项目目录**

```json
{
  "mcpServers": {
    "investTemplate": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/shuai.liang/工作/Private/investTemplate"
      ]
    }
  }
}
```

**步骤 3：使用**

```
在 Claude Desktop 对话框：

你：读取 AGENTS.md 文件
Claude：（读取项目指南）

你：帮我分析工商银行（01398.HK）
Claude：（按照标准流程生成报告）
```

### 方式 3：Claude.ai（网页版）

**限制**：无法直接访问本地文件

**解决方案**：
1. 复制 `AGENTS.md` 内容
2. 粘贴到对话框
3. 然后提问

```
你：这是我的项目指南（粘贴 AGENTS.md）

现在帮我分析工商银行...
```

### 方式 4：Claude API（编程接口）

**适合场景**：自动化脚本

```python
import anthropic

client = anthropic.Anthropic(api_key="your-api-key")

# 读取 AGENTS.md
with open("AGENTS.md", "r") as f:
    agents_content = f.read()

# 发送请求
message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=8000,
    system=agents_content,  # 将 AGENTS.md 作为系统提示
    messages=[{
        "role": "user",
        "content": "帮我分析工商银行（01398.HK）"
    }]
)
```

## AGENTS.md 指南

### AGENTS.md 是什么？

这是一个**专门写给 AI 的使用指南**，包含：

1. **项目概况**：类型、目标用户、投资市场
2. **核心文件**：哪些文件最重要
3. **操作规范**：如何生成报告、如何更新数据
4. **数据源等级**：S/A/B/C/D 级的定义
5. **禁止事项**：AI 不能做什么
6. **常见错误案例**：前车之鉴

### 关键内容

#### 1. 数据源等级（强制执行）

```
S级：公司年报原文 → 必须使用
A级：业绩公告 → 需验证
B级：东方财富等 → 不得用于核心数据
C级：财经媒体 → 仅供参考
D级：AI估算 → 禁止用于投资决策
```

#### 2. 生成报告的标准流程

```
1. 读取 个股分析标准模版.md
2. 使用 S级数据源（年报原文）
3. 保存到 07-分析输出/{公司}_{代码}_投资分析报告.md
4. 更新 .vitepress/config.mjs 侧边栏
5. 更新 07-分析输出/index.md 目录索引
6. 更新 index.md 首页列表
```

#### 3. 首行必须格式

```markdown
**一句话结论**：🐢🍊 极品金龟，PB 0.4倍，股息率8%，建议买入
```

#### 4. 模拟持仓的 4 文件同步

```
⚠️ 修改持仓时必须同时更新 4 个文件：

1. 08-决策追踪/simulation_trades.csv      ← 唯一真相源
2. 08-决策追踪/simulation_state.json       ← 运行时状态
3. 08-决策追踪/dashboard_snapshot.json     ← 决策追踪
4. public/dashboard/dashboard_snapshot.json ← 网页数据源（易遗漏！）
```

#### 5. 禁止事项

```
❌ 使用 D级数据（AI估算）用于投资决策
❌ 为同一公司保留多份历史版本报告
❌ 直接修改 state.json 而不更新 trades.csv
❌ 修改持仓数据时遗漏 public/dashboard/ 目录
❌ 自动执行 git commit/push（必须由用户确认）
```

## 注意事项

### 1. 数据源的重要性 ⭐⭐⭐⭐⭐

```
S级数据（年报原文）→ 必须使用
D级数据（AI估算）→ 禁止使用
```

**错误示例**：
```
❌ AI："根据估算，工商银行2023年净利润约300亿"
✅ AI："根据2023年年报，工商银行净利润为341.23亿"
```

### 2. 4 文件同步 ⭐⭐⭐⭐⭐

修改持仓数据时，**必须**同时更新 4 个文件：

```
simulation_trades.csv           ← 源头
simulation_state.json           ← 计算
08-决策追踪/dashboard_snapshot.json
public/dashboard/dashboard_snapshot.json  ← 容易遗漏！
```

### 3. 验证数据一致性

```bash
# AI 应该自动运行这个验证
python scripts/validate_simulation_data.py
```

### 4. 首行格式固定

```markdown
**一句话结论**：🐢🍊 极品金龟，PB 0.4倍，股息率8%，建议买入
```

不能写成：
```markdown
❌ 结论：建议买入
❌ 总结：这是一只好股票
```

### 5. Git 操作需用户确认

```
❌ AI 不能自动执行：
   git commit -m "..."
   git push

✅ AI 应该：
   提示你需要手动 commit
```
